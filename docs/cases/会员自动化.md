# 会员自动化

## 项目背景

会员自动化项目由高高提出，启动背景是门店收银流程混乱、操作方式不统一，同时原有处理步骤复杂。一线在开卡、升级、退卡、优惠等场景中需要做大量人工操作，不同同事的处理方式也不一致。

在领健系统中，会员、卡券权益、收费彼此独立，缺少自动对应关系。对一线来说，这意味着收银流程繁琐、容易出错；对财务来说，这意味着月底核对收入、优惠和会员权益时需要大量人工比对，管理成本高。

## 项目目标

- 简化一线操作时长，提升效率。
- 规范会员优惠流程，避免不必要的折扣方式。
- 建立会员、卡券、收费、账单之间的自动对应关系。
- 降低财务核对和审查成本。

## 我的角色

我在这个项目中同时承担产品设计、产品经理和项目推进者的角色。

我主要负责：

- 设计方案输出。
- 业务逻辑与规则确认。
- 推动开发进度。
- 上线后会员优惠券的维护与梳理。
- 上线后的运营跟进与问题修改。

## 核心问题

- 会员收银流程不规范，原有优惠审批形同虚设，既增加操作步骤，也缺少真正有效的约束边界。
- 一线高频操作步骤多、易错，跨门店升级、退卡、年龄切卡等场景处理成本高。
- 财务需要人工核对会员、优惠券、收费之间的关系，月底管理成本高，也难以快速识别操作是否违规。

## 关键方案

### 规则系统化

项目最核心的工作，不是增加几个会员功能，而是重新梳理会员卡、卡券、收费、优惠、账单之间的关系与对应规则，让原本分散的对象被系统统一承接。

单年卡、长期卡、囤卡、优惠券、手动优惠、跨门店账单、年龄自动切卡等复杂场景，都需要建立一套统一规则，才能真正解决前台、运营和财务在使用中的混乱问题。

### 流程自动化

在规则被明确之后，项目进一步把开卡、升续卡、退卡、冻结、囤卡等高频流程从多步人工操作改成系统自动承接，减少一线处理时间与误操作。

这类自动化不只是减少点击步骤，更重要的是把过去依赖人工记忆和补操作的动作，交给系统完成，从而降低流程断裂和信息遗漏。

### 合规与财务治理

项目没有继续沿用低效的优惠审批方式，而是引入优惠额度，在保证一线操作自由度的同时，建立更明确的优惠边界。

通过系统留痕和自动对应关系，会员、优惠券、收费之间开始形成可核对的结构，减少违规操作，也降低财务月底人工核对成本。

## 推进与落地

- 前期调研 2 周。
- 开发周期采用 `2+1` 模式，即 2 周开发、1 周测试。
- 先在试点门店试用约 2 周，再全量上线。
- 上线后继续跟进运营使用问题并迭代修改。
