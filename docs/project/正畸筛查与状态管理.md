# 正畸筛查与状态管理

## 项目背景

在儿童口腔场景中，矫正前置管理长期缺少一套清晰、连续的流程。过去更多依赖各门店自行组织面评会，由正畸医生主导，儿牙医生主要承担患者信息传递的角色。流程本身不够清晰，角色参与度不均衡，也缺少对筛查、结论、报告和后续预约的连续管理。

## 我的角色

该项目中的流程梳理、逻辑设计和功能拆分主要由我主导完成。我不仅参与页面和交互层设计，也参与了更上游的业务问题识别、流程重构、角色职责拆分与状态规则设计。

## 主要工作

- 梳理原有面评和筛查流程中的核心问题，包括重复筛选、临时患者遗漏、历史数据不准、多人分步协作支持不足等。
- 重构儿牙、正畸、运营等角色之间的协作方式，明确不同角色在流程中的职责边界。
- 设计筛查推荐逻辑、再次筛查逻辑和矫正状态规则，使筛查过程从一次性动作转向持续管理。
- 拆分不同角色在系统中的任务、可见信息与可执行操作，覆盖护士、儿牙医生、正畸医生、运营等角色。
- 梳理与筛查相关的功能结构，包括状态展示、待办看板、筛查表与正畸方案衔接、报告生成与反馈等内容。

## 解决的问题

这项工作的核心不是单一页面设计，而是把原本分散、模糊、依赖人工经验的筛查过程，整理为一套更清晰的矫正上漏斗流程。项目重点处理了以下问题：

- 原有面评与快筛之间存在重复操作。
- 临时患者与历史数据场景下，容易出现筛查遗漏。
- 儿牙医生和正畸医生多人分步协作时，系统支持不足。
- 不同角色在流程中的参与边界不清晰，儿牙医生参与度偏低。
- 缺少统一的推荐、再筛查和后续跟进逻辑。

## 项目结果

这项工作的一个重要结果，是推动建立了一套更清晰的矫正管理上漏斗流程。过去这部分相对缺失，更多依赖面评会和正畸医生个人推动；调整后，儿牙医生在筛查环节中的参与被前移，也更系统地纳入了流程之中。

这套流程不仅服务于内部协作，也让矫正管理开始具备连续的数据指引。围绕筛查、结论、报告、阅读和预约，团队可以持续看到关键节点的转化情况，使矫正管理过程变得更可控、更清晰。

从患者侧看，这也对应了一条更明确的咬合管理路径，让儿童会员能够更早发现问题，并进入后续服务流程。
